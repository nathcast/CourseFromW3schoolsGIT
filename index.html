I may as well take notes here.

  • git init: to initalise environemnt
  • git status: checks where we are at
  • git add <file> : add file to stage area
  • git add --all or git add -A: add all files to staging areas
  • git commit -m "First release of the Git application": Adding commits keep track of our progress and changes as we work.
  • git commit --help: opens help page on browser (-help: local help)
  • git help -g : lists the concepts
  • git branch hello-world: creates a branch
  • git branch: lists the branches
